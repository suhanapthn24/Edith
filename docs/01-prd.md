# APEX — Product Requirements Document
### v1.1 · June 2026

## Vision
APEX is a personal AI operating system that functions as a chief of staff — aware of your goals, schedule, learning progress, and projects, and capable of reasoning across all of them to continuously answer: *"What should I do right now?"*

## Design Principles
| Principle | Meaning |
|---|---|
| **AI-first** | Every feature is backed by an agent, not just a form |
| **Local-first AI** | Use Ollama (free, local) by default; Claude only for complex tasks |
| **Context-aware** | The system knows your full state before making suggestions |
| **Progressive** | Starts simple, learns your patterns, grows with you |
| **Integrated** | Pulls from real external data (calendar, LeetCode, arXiv) |

---

## Module Specifications

### M1 — AI Life Dashboard
- **Purpose:** Single pane of glass for daily orientation
- **Inputs:** All other modules' state
- **Outputs:** Daily priority list, focus recommendation, streak summary, mini briefing
- **AI capability:** Natural language daily briefing generated by Orchestrator Agent (Ollama)
- **Key metrics:** task completion rate, streak length, goal progress %, study hours/day

### M2 — Calendar Intelligence
- **Purpose:** Smart scheduling that fills free time with purposeful work
- **Inputs:** Google Calendar, Outlook, user preferences
- **Outputs:** Optimized schedule, smart event creation, conflict detection
- **AI capability:** Detects free blocks, suggests activity type based on context
- **Key metrics:** free-block utilization rate, schedule adherence

### M3 — Language Learning Hub
- **Purpose:** Structured multilingual progression engine
- **Languages:** French, Arabic, Mandarin (active); Italian, Spanish (planned)
- **Inputs:** User session data, vocabulary tests, spaced repetition logs
- **Outputs:** Daily lessons, weekly curriculum, flashcard decks, AI tutor conversations
- **AI capability (Ollama):** Adapts lesson difficulty, generates contextual sentences, conversational practice
- **AI capability (Claude):** Complex grammar explanations, curriculum design
- **Key metrics:** vocabulary retained, CEFR level progress, daily streak, accuracy %

### M4 — Research Paper Hub
- **Purpose:** Systematic literature review and knowledge extraction engine
- **Sources:** arXiv, Semantic Scholar, user-uploaded PDFs
- **Inputs:** Papers (PDF or URL), user annotations
- **Outputs:** AI summaries, methodology extraction, dataset extraction, implementation roadmap
- **AI capability (Claude):** Long-context PDF processing, structured extraction (requires 200K context)
- **AI capability (Ollama):** Simple Q&A about already-summarized papers
- **Key metrics:** papers read, notes density

### M5 — DSA Learning System (NeetCode 150)
- **Purpose:** Competitive programming skill tracker following NeetCode 150 roadmap
- **Curriculum:** NeetCode 150 problems organized by their official category groupings
- **NeetCode Categories:**
  - Arrays & Hashing (9 problems)
  - Two Pointers (5)
  - Sliding Window (6)
  - Stack (7)
  - Binary Search (7)
  - Linked List (11)
  - Trees (15)
  - Tries (3)
  - Heap / Priority Queue (7)
  - Backtracking (9)
  - Graphs (13)
  - Advanced Graphs (6)
  - 1-D Dynamic Programming (12)
  - 2-D Dynamic Programming (11)
  - Greedy (8)
  - Intervals (6)
  - Math & Geometry (8)
  - Bit Manipulation (7)
- **Integration:** LeetCode (link to problem, track solve status)
- **AI capability (Ollama):** Explain approach, hint generation, complexity analysis
- **AI capability (Claude):** Deep solution walkthrough, pattern analysis across problems
- **Key metrics:** NeetCode 150 completion %, category mastery, revision due count

### M6 — Knowledge Vault
- **Purpose:** Personal knowledge base with semantic search
- **Storage:** Notes, PDFs, URLs, videos, articles, bookmarks
- **Technology:** pgvector embeddings, RAG pipeline
- **AI capability (Ollama):** Answer questions using stored knowledge (local RAG)
- **AI capability (Claude):** Complex synthesis across many documents
- **Key metrics:** items stored, query satisfaction

### M7 — Career Operating System
- **Purpose:** Job search and career development tracker
- **Tracks:** Resume versions, applications, interviews, companies, networking contacts
- **AI capability (Ollama):** Basic skill extraction, application status suggestions
- **AI capability (Claude):** Deep skill gap analysis, resume tailoring, interview prep
- **Key metrics:** applications sent, response rate, interview conversion

### M8 — Skill Building & Certifications
- **Purpose:** Structured skill development tracker with learning path management
- **Tracks:**
  - Active certifications (AWS, GCP, Azure, TensorFlow Developer, etc.)
  - Courses in progress (Coursera, Udemy, edX, fast.ai, etc.)
  - Books being read (technical + non-technical)
  - Tutorials and workshops
  - Skills being developed (languages, frameworks, tools)
- **Stores per certification/course:**
  - Provider, URL, expected duration, deadline
  - Progress % and current chapter/section
  - Notes and key takeaways
  - Status: planned / in_progress / completed / paused
- **AI capability (Ollama):** Recommend next course based on goals, study schedule generation
- **AI capability (Claude):** Career-aligned learning path design, certification priority scoring
- **Key metrics:** certifications earned, courses completed, study hours logged, skills added

## Non-Functional Requirements
| Category | Requirement |
|---|---|
| **Performance** | Dashboard load < 2s, AI response < 5s (Ollama), < 8s (Claude) |
| **Availability** | 99.5% uptime (personal tool) |
| **Security** | OAuth 2.0, JWT tokens, encrypted PII at rest |
| **Privacy** | Ollama runs locally — sensitive data never leaves machine; Claude calls are opt-in |
| **Cost** | Minimize Claude API calls — Ollama handles 80%+ of agent work |
